## 1. 법적 · 사업자 (최우선 — 이미 별도 추적 중, #162)
- [ ] 사업자등록
- [ ] 통신판매업 신고 (구청) — 제휴 예약 수익이 발생하기 전에 필요
- [ ] 위치기반서비스 신고 필요 여부 검토 (방통위) — 단순 지도 표시만이면
      대상이 아닐 수 있음, 실시간 위치 수집 기능 범위 확정 후 판단

## 2. Kakao / Google 콘솔 설정

Kakao Developers 콘솔은 서로 다른 설정 3곳이 있고, 하나만 빠져도 각각 다른
증상으로 나타납니다 — 헷갈리기 쉬워서 구분해서 적습니다.

- [ ] **① 카카오 로그인 > Redirect URI** — 소셜 로그인 콜백 주소
      (`https://<도메인>/api/auth/callback/kakao`)가 정확히 등록돼 있는지.
      빠지면 로그인 자체가 실패하거나 세션이 깨짐.
- [ ] **② 내 애플리케이션 > 플랫폼 설정 > Web 사이트 도메인** — 지도(JS SDK)와
      **카카오톡 공유하기** 둘 다 이 설정 하나에 걸려 있음. 여기에 실제
      서비스 도메인(`https://www.tradule.co.kr`, 프리뷰로도 테스트한다면
      `https://travel-xi-red.vercel.app`도)이 정확히 등록돼 있는지 확인.
      **미등록 시 실제로 재현된 증상**: "카카오톡으로 트래블 메이트
      초대하기" 버튼을 눌렀을 때 카카오톡 공유 창 대신 빈
      `about:blank` 팝업만 뜸 — 코드는 정상 호출까지는 하는데, 카카오
      서버가 미등록 도메인의 요청을 거부하면서 팝업에 아무것도 못
      띄우는 상태. 이건 코드로 고칠 수 없는 콘솔 설정 문제라, 이 항목
      확인이 필요합니다.
- [ ] **③ REST API 키(국내 장소 검색용)** — ①②와 별개의 키. 도메인/쿼터
      제한이 걸려 있는지 확인
- [ ] 위 두 번째 항목(트메 신청 버튼 무반응) 자체는 이미 코드로 원인을
      찾아 고쳤음(PR #90 — 서버 응답 실패를 조용히 무시하던 클라이언트
      버그였음). 그것과 이번 "카카오톡 공유 빈 팝업"은 증상은 비슷해
      보여도 원인이 다른 별개 문제입니다.
- [ ] Google Maps Platform — 결제 계정 연결, API 키를 HTTP 리퍼러/서버 IP로
      제한, 사용하는 API만 활성화(Maps JS, Places API New), 예산 알림 설정
- [ ] Google/Kakao/Apple OAuth 프로덕션 redirect URI가 실제 도메인 기준으로
      등록돼 있는지 재확인 (Apple은 앱 검수가 필요할 수 있음)

## 3. Vercel 환경변수 점검
- [ ] `NEXTAUTH_URL`(또는 Auth.js가 쓰는 호스트 값)이 실제 프로덕션 도메인과
      일치하는지 확인 — 다르면 세션 쿠키가 예상과 다르게 동작할 수 있음
- [ ] `AUTH_SECRET`, 각 provider의 ID/SECRET이 프로덕션 값으로 채워져 있는지
- [ ] `DATABASE_URL`이 실제 운영 DB를 가리키는지, 커넥션 제한(max)이 그 DB
      플랜에 맞게 설정돼 있는지

## 4. 에러 모니터링 서비스 가입
- [ ] Sentry(또는 대체 서비스) 계정 생성 + 프로젝트 생성 + DSN 발급
      → DSN을 Vercel 환경변수로 넣어주시면, SDK 연동 코드는 제가 넣습니다
- [ ] (선택) Uptime 모니터링 서비스(Better Uptime 등) 가입 — 서비스 다운을
      사용자보다 먼저 알기 위함

## 5. DB 백업
- [ ] 현재 쓰고 있는 Postgres 호스팅사(Neon/Supabase/RDS 등) 확인
- [ ] 그 호스팅사의 자동 백업 기능 켜기 + 보관 기간 확인
- [ ] 복구 절차를 실제로 한 번 테스트해보기 (문서 어디에도 아직 이 답이 없음)

## 6. 앱스토어 패키징(#115) 자체를 위한 사전 준비
- [ ] Apple Developer Program 계정 (연 $99) — iOS 배포 필수
- [ ] Google Play Console 계정 (1회 $25) — Android 배포 필수
- [ ] 앱 아이콘 세트 · 스크린샷 · 스토어 설명 문구 (한국어 기준 우선)
- [ ] 스토어 등록에 필요한 개인정보처리방침 **공개 URL** 확정
      (`/privacy` 페이지는 이미 있음 — 커스텀 도메인 URL로 스토어에 제출)
- [ ] Apple 심사 필수 항목 재확인: 로그인한 계정의 앱 내 탈퇴(계정 삭제)
      경로 — 이미 구현됨(`ProfileSheet` 회원 탈퇴), 심사 시 노출 경로만 확인
- [ ] 제휴 예약 딥링크의 인앱 브라우저 처리 방식 결정 — TWA/Capacitor로
      패키징하면 Custom Tabs / SFSafariViewController를 쓸 수 있음(현재 PWA는
      불가능). "네이티브 패키징 때 같이 처리" 로 이미 합의됨, 이 작업 시작할
      때 다시 논의
