# Tradule 상용화 가이드라인 (웹 · 앱)

프로토타입을 실제 서비스로 출시하기 위해 필요한 항목을 영역별로 정리했습니다.
현재 코드베이스 기준으로 "이미 된 것 / 해야 할 것"을 구분했고, **★ = 출시 전
필수**, ☆ = 출시 후 개선 권장입니다.

---

## 0. 현재 상태 요약
- ✅ Next.js(App Router) + Postgres 스키마 + Auth.js(구글/카카오/애플) 골격
- ✅ Google Places / Kakao Local 실검색, 지도(구글/카카오) 추상화
- ✅ 숙소 제휴 링크 메커니즘(승인 후 env만 채우면 활성화), PWA manifest, 스플래시
- ⚠️ 대부분의 추천/트렌드 데이터는 **큐레이션된 플레이스홀더** — 실데이터 파이프라인 연결 필요
- ⚠️ 결제/구독 없음, 분석/모니터링 없음, 법적 문서(약관·개인정보) 없음

---

## 1. 법적 · 정책 (★ 최우선, 국내 기준)
- **★ 사업자등록**: 수익(제휴/광고) 발생 시 필요. 온라인 사업이면 **통신판매업 신고**(구청)도 대개 필요.
- **★ 개인정보처리방침**: 소셜로그인으로 이름/이메일을 수집하므로 필수. 수집 항목·목적·보관기간·파기·제3자 제공(제휴사 이동 포함)·이용자 권리·문의처 명시.
- **★ 이용약관**: 서비스 범위, 책임 한계(추천/평점은 외부 데이터), 계정/콘텐츠 정책.
- **★ 제휴/광고 표시**: 제휴 링크에 "제휴" 고지(이미 UI에 반영). 공정위 표시광고 및 각 제휴사 약관 준수.
- **★ 위치정보**: 사용자 실시간 위치를 수집·이용하면 **위치정보사업/위치기반서비스 신고**(방통위) 검토. 단순 지도 표시만이면 대상 아님 — 기능 확정 후 판단.
- **★ 오픈소스/지도 약관**: Google Maps Platform · Kakao 지도 약관의 로고/링크 표기, 캐싱 제한 준수. Google 리뷰/사진은 자체 저장 금지(딥링크로 연결 — 이미 그렇게 구현).
- **☆ 해외 이용자**: GDPR(EU)·CCPA(미국) 대응(동의·삭제요청 처리)은 해외 타깃 시.
- **☆ 만 14세 미만**: 국내법상 법정대리인 동의 이슈 — 가입 연령 정책 정하기.

## 2. 인프라 · 배포 (★)
- **★ 호스팅**: Vercel(현재 배포처)로 충분. 프로덕션/프리뷰 환경 분리.
- **★ 커스텀 도메인 + HTTPS**: 도메인 구입 → Vercel 연결(자동 TLS).
- **★ 관리형 Postgres**: Neon / Supabase / RDS 등. `DATABASE_URL` 설정 후 `npm run db:migrate`. 자동 **백업** 켜기.
- **★ 시크릿 관리**: 모든 키를 Vercel 환경변수로(리포에 커밋 금지). `.env.example`이 필요한 키 목록 — `AUTH_SECRET`은 `npx auth secret`로 생성.
- **☆ 스테이징 환경**: 프로덕션 배포 전 검증용 별도 환경.

## 3. 외부 API 키 · 쿼터 · 비용 (★)
- **★ Google Maps Platform**: 결제 계정 연결, **API 키 제한**(HTTP 리퍼러/서버 IP), 사용 API만 활성화(Maps JS, Places API New). **예산 알림 + 일일 쿼터** 설정으로 비용 폭주 방지.
  - 주의: 실검색이 검색당 **여러 번** 호출됨(팬아웃 + 원어 병기). 트래픽 늘면 캐싱(현재 5분 클라이언트 캐시) 외에 **서버 캐시/일 쿼터**로 관리.
- **★ Kakao**: JavaScript 키(지도)와 REST 키(로컬 검색) 분리 발급, 도메인/쿼터 설정. (국내 지도용 실 키 필요 — 현재 미설정)
- **★ 제휴 프로그램 승인**: 아고다/트립/호텔스/야놀자/여기어때 — 승인 후 `NEXT_PUBLIC_*` env 채우면 자동 활성화(코드 완료됨). 정확한 파라미터명은 각 대시보드 확인.
- **☆ 실데이터 파이프라인**: 트렌드/추천을 실제 소스로 교체(`src/server/pipeline`), 플레이스홀더 제거.

## 4. 인증 · 계정 (★)
- **★ OAuth 프로덕션 설정**: 각 provider 콘솔에 **프로덕션 redirect URI**(`https://<도메인>/api/auth/callback/<provider>`) 등록. 애플/구글은 앱 검수 필요할 수 있음.
- **★ `AUTH_SECRET`, provider ID/SECRET** 프로덕션 값 설정.
- **★ 계정 삭제/데이터 다운로드**: 개인정보법상 파기·열람 요청 처리 경로(최소 이메일 문의라도).
- **☆ 세션 정책**: 만료/재발급, 다중 기기.

## 5. 수익화 (☆→★)
- **★(수익화 시) 제휴 고지 + 링크 규격**: 1장 참조. 클릭/전환 추적을 붙이면 수익 분석 가능.
- **☆ 광고**: 필요 시 디스플레이 광고, 단 UX/정책 균형.
- **☆ 구독/PG**: 프리미엄 기능 유료화 시 국내 PG(토스페이먼츠 등) 또는 앱스토어 인앱결제(스토어 정책상 디지털재화는 인앱결제 강제).

## 6. PWA · 앱스토어 (☆)
- ✅ `manifest.ts` + 스플래시 + theme-color 골격 완료.
- **★(PWA로 낼 경우) 아이콘 애셋**: `public/brand/` README의 사이즈대로 채우기.
- **☆ 스토어 출시**: 웹을 그대로 앱으로 — Android는 **TWA(Bubblewrap)**, iOS/양쪽은 **Capacitor** 래핑. 스토어 심사(개인정보 라벨, 스크린샷, 로그인 시 계정삭제 경로 = 애플 필수) 준비.

## 7. 분석 · 모니터링 (☆, 출시와 함께 권장)
- **☆ 제품 분석**: GA4 또는 Amplitude/PostHog — 검색·저장·제휴클릭 전환 추적.
- **☆ 에러 모니터링**: Sentry(프론트+API 라우트).
- **☆ 가용성**: Uptime 모니터(Better Uptime 등) + Vercel 로그/알림.

## 8. 성능 · SEO · 접근성 (☆)
- **☆ Lighthouse**: LCP/CLS 점검. 지도/이미지 지연로딩은 이미 적용.
- **☆ SEO**: `metadata`에 OG/트위터 카드, `sitemap.ts`·`robots.ts`, 공유 미리보기 이미지(`og-image.png`).
- **☆ 접근성**: 대비(밝은 회색 텍스트 상향), 포커스 링, 터치 타깃 44px — 디자인 리뷰 항목 반영.

## 9. 보안 (★ 일부)
- **★ 시크릿 유출 방지**: 리포 스캔, `NEXT_PUBLIC_*`에는 공개 가능한 값만.
- **★ API 라우트 보호**: 쓰기 라우트 인증/소유권 체크, 공유 토큰 추측 방지.
- **☆ Rate limiting**: 검색/공유 API 남용 방지(Upstash 등).
- **☆ 의존성 취약점**: `npm audit`/Dependabot, CSP 헤더.

## 10. 운영 · 지원 (☆)
- **☆ 고객 문의 창구**(이메일/폼), 약관 동의 시점 로깅, 데이터 삭제 요청 처리 절차.
- **☆ 상태 페이지/공지**, 장애 대응 런북.
